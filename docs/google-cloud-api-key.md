# Google Cloud API キー取得手順（Translation）

このアプリは **Google Cloud Translation API** を使用して、テキストの **言語判定** と **日英翻訳** を行います。  
（音声→テキストはブラウザの Web Speech API を使用します）

## 前提

- Google アカウント
- 課金の有効化が必要になる場合があります（組織の方針に従ってください）

## 1. Google Cloud プロジェクトを作成

1) Google Cloud Console を開く  
2) プロジェクト セレクタから **新しいプロジェクト** を作成  
3) 組織（Organization）/フォルダ/請求先アカウントを必要に応じて設定

## 2. Translation API を有効化

1) 対象プロジェクトを選択  
2) 「API とサービス」→「ライブラリ」  
3) **Cloud Translation API** を検索して **有効にする**

## 3. API キーを作成

1) 「API とサービス」→「認証情報」  
2) 「認証情報を作成」→ **API キー**  
3) 作成された API キー（`AIza...` から始まることが多い）を控える

## 4.（推奨）API キーを制限する

企業利用では、API キーをそのまま使うのではなく、可能な範囲で制限してください。

### 4-1. API の制限

1) 「認証情報」→ 作成した API キーを開く  
2) 「API の制限」→ **キーを制限**  
3) **Cloud Translation API** のみにチェックして保存

### 4-2. アプリケーションの制限（目安）

このアプリはローカルで `http://127.0.0.1:5173` で動作します。

- 「アプリケーションの制限」→（可能であれば）**HTTP リファラ** を選択  
  - 許可するリファラ例: `http://127.0.0.1:5173/*`  
  - 会社のプロキシ/ブラウザ方針によっては挙動が異なるため、管理者と相談してください

※ API キーの露出リスクを下げるには、本来は **ブラウザから直接 API キーを扱わない構成（サーバ側で保護）** が望ましいです。  
このリポジトリの実装は「ローカルアプリとして最小構成で動かす」ことを優先しています。

## 5. アプリに API キーを設定

1) アプリを起動（`npm start`）  
2) 画面左の「Google Cloud API キー」に貼り付け  
3) 「保存」→「接続テスト」  
4) 問題なければ「開始」

## よくあるエラー

- `APIキーが不正です`  
  - 余分な空白が混ざっていないか確認
- `This API method requires billing to be enabled`（課金が必要）  
  - プロジェクトで請求先アカウントを設定
- `Requests to this API ... are blocked`（制限でブロック）  
  - API 制限/リファラ制限の設定を見直す
